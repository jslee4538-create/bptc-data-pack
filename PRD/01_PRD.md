## 1. 제품 개요

### 한 줄 요약
교토에서 교환학생 생활을 하는 청년들이 수업 외 여가 시간을 알차게 보낼 수 있도록 관광지, 맛집, 계절 행사, 버스 노선을 한 곳에서 안내하는 모바일 웹앱.

### 해결하는 문제
교토에 처음 온 교환학생들은 어디를 가야 할지, 어떤 행사가 있는지, 버스를 어떻게 타야 하는지 모른다. 정보가 여러 사이트(교토 관광청, 구글맵, 블로그)에 흩어져 있고, 대부분 일본어라 접근성이 떨어진다.

### 핵심 가치
- **교환학생 맞춤**: 일반 관광객이 아닌 6개월 체류 교환학생의 생활 패턴에 맞춤
- **올인원 가이드**: 관광지 + 맛집 + 계절 행사 + 버스 노선을 한 앱에서 해결
- **계절 중심**: 교토의 사계절 행사를 놓치지 않도록 캘린더로 안내

---

## 2. 사용자

### 주요 사용자
- **누구**: 교토 소재 대학에 교환학생으로 오는 20대 청년 (국적 불문)
- **상황**: 수업 후 여가 시간, 주말, 방학 중 교토와 주변을 탐험할 때
- **목표**: 현지인만 아는 숨은 명소, 제철 행사를 발견하고, 버스로 효율적으로 이동하기

### 사용자 시나리오
1. 도시샤 대학교에 교환학생으로 온 김민수(25세)가 주말에 앱을 열고
2. "가을 단풍" 카테고리에서 추천 장소를 탐색한 뒤
3. 도후쿠지를 즐겨찾기에 저장하고, 버스 노선 조회로 가는 방법을 확인한다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
소셜 로그인 -> 홈(추천 장소 + 다가올 행사) -> 장소 상세 -> 즐겨찾기/버스 조회
```

### 상세 흐름
1. **소셜 로그인**: Google 계정으로 1초만에 가입/로그인
2. **홈 화면**: 추천 관광지/맛집 카드와 다가오는 계절 행사가 한눈에 보임
3. **카테고리 탐색**: 신사/사찰, 맛집, 카페, 자연, 문화체험 등 카테고리별 필터링
4. **장소 상세**: 사진, 설명, 주소, 운영시간, 가격대, 태그 확인
5. **즐겨찾기 저장**: 하트 버튼으로 저장, 마이페이지에서 모아보기
6. **캘린더 확인**: 월별/계절별 행사 목록, 행사 상세 및 연결 장소 확인
7. **버스 조회**: 노선 번호로 검색하거나 지도에서 정류장 탭하여 경유 노선 확인

## 5. 성공 기준

- [ ] 교환학생 10명이 실제로 1주일간 사용하고 "유용하다"고 답변
- [ ] 관광지/맛집 50개 이상, 계절 행사 20개 이상 등록
- [ ] 버스 노선 주요 10개 이상, 정류장 100개 이상 등록
- [ ] 모바일에서 3초 이내 페이지 로딩
- [ ] Lighthouse 성능 점수 90점 이상

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 실시간 버스 도착 정보 -- 이유: 교토시 개방 API 연동 필요, Phase 3에서 검토
- AI 맞춤 추천 -- 이유: 외부 API 비용 발생, Phase 3에서 검토
- 네이티브 앱 (iOS/Android) -- 이유: 모바일 웹앱으로 충분, PWA로 대체
- 호텔/숙소 예약 -- 이유: 교환학생은 대학 기숙사 사용, 예약 기능 불필요
- 결제/이커머스 -- 이유: 정보 제공 앱이지 상거래 앱이 아님
- 관리자(Admin) 대시보드 -- 이유: MVP에서는 Supabase 콘솔로 직접 관리
- 사용자 간 채팅/DM -- 이유: 커뮤니티 기능은 Phase 3에서 검토
