# 교토 여가 가이드 (Kyoto Leisure Guide) -- Phase 분리 계획

> 한 번에 다 만들면 복잡해져서 품질이 떨어집니다.
> Phase별로 나눠서 각각 "진짜 동작하는 제품"을 만듭니다.

---

## Phase 1: MVP (2~3주)

### 목표
교토의 관광지/맛집을 탐색하고, 계절 행사를 확인하고, 버스 노선을 조회할 수 있는 실제 동작하는 모바일 웹앱

### 기능
- [ ] 소셜 로그인 (Google)
- [ ] 관광지/맛집 목록 페이지 (카드 형태)
- [ ] 카테고리별 필터링 (신사, 맛집, 카페 등)
- [ ] 장소 상세 페이지 (사진, 설명, 주소, 운영시간, 가격대, 태그)
- [ ] 계절별 행사/이벤트 캘린더 (월별 보기)
- [ ] 행사 상세 페이지 (설명, 기간, 연결 장소)
- [ ] 즐겨찾기 (장소/행사 저장 및 마이페이지에서 모아보기)
- [ ] 버스 노선 조회 (노선 번호로 검색)
- [ ] 버스 정류장 지도 (Leaflet + OpenStreetMap)
- [ ] 반응형 모바일 UI (Tailwind CSS)

### 데이터
- User, Place, Category, Event, BusRoute, BusStop, Bookmark

### 인증
- Supabase Auth + Google OAuth

### "진짜 제품" 체크리스트
- [ ] 실제 DB 연결 (목업 데이터 X)
- [ ] 실제 인증 (하드코딩된 비밀번호 X)
- [ ] 실제 서버에 배포 (localhost X)
- [ ] 다른 사람이 URL로 접속해서 써볼 수 있음
- [ ] 관광지 최소 50개, 행사 최소 20개, 버스 노선 최소 10개 등록

## Phase 2: 확장 (2주)

### 전제 조건
- Phase 1이 안정적으로 배포된 상태

### 목표
검색, 장소 기반 버스 길찾기, 행사 알림, 나만의 코스 기능으로 사용성을 한 단계 높임

### 기능
- [ ] 검색 기능 (장소명 / 태그 / 지역으로 통합 검색)
- [ ] 장소 기반 버스 길찾기 ("이 장소에 가려면 몇 번 버스?")
- [ ] 행사 알림 (D-7, D-1 브라우저 알림)
- [ ] 나만의 코스 만들기 (장소 여러 개를 하루 일정 순서로 묶기)
- [ ] 다국어 지원 (한국어 / 영어 / 일본어)

### 추가 데이터
- Course (코스): id, user_id, title, description, created_at
- CourseItem (코스 항목): id, course_id, place_id, order, memo

### 통합 테스트
- Phase 1 기능(로그인, 장소 조회, 즐겨찾기, 캘린더, 버스 지도)이 여전히 정상 동작하는지 확인

## Phase 3: 고도화 (2~3주)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
AI 추천, 오프라인 지원, 커뮤니티 기능으로 교환학생 필수 앱이 됨

### 기능
- [ ] AI 맞춤 추천 (취향 / 날씨 / 현재 위치 기반)
- [ ] 오프라인 지도 캐시 (인터넷 없이 지도 사용 가능)
- [ ] 교환학생 후기/꽁팁 커뮤니티 (장소별 후기 작성 및 열람)
- [ ] 실시간 버스 도착 정보 연동 (교토시 개방 API)
- [ ] PWA 설치 (홈화면 추가 + 앱처럼 실행)

### 추가 데이터
- Review (후기): id, user_id, place_id, content, rating, created_at
- UserPreference (취향): id, user_id, preferred_categories, budget_level

### 주의사항
- AI 추천: OpenAI 또는 Claude API 비용 발생 가능 (월 사용량에 따라 $10~50)
- 실시간 버스: 교토시 버스 개방 API 사용 가능 여부 확인 필요
- PWA: Service Worker 캐싱 전략 설계 필요
