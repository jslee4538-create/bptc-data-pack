# AI에게 SQL 시키기 — 프롬프트 템플릿

> 강의 수강생용. Claude / ChatGPT 어디서나 동작.
> 핵심 메시지: **"SQL은 외우는 게 아니라 AI에게 시키는 것"**

## 🎯 가장 안전한 4단계 프롬프트

```
1단계 (구조 파악)
  "CLAUDE.md 파일을 읽고 이 데이터베이스 구조를 파악해줘.
   부산 부동산 DB로 질문할 거야."

2단계 (질문)
  "해운대구 2025년 매매 TOP 10 단지를 평균 거래금액 순으로 보여줘."

3단계 (제약)
  "- dealAmount는 만원 단위니까 억원으로 변환
   - LIMIT 10
   - 한글 컬럼명은 큰따옴표로 감싸기"

4단계 (검토)
  "이 SQL 한 줄씩 비개발자가 이해 가능하게 한글로 설명해줘."
```

## 시나리오별 프롬프트 5종

### 1. 부산 도시철도 — 본인 자주 가는 역 시간대 패턴

```
부산 도시철도 데이터(bptc_busan.db)에서 
"서면역 평일 시간대별 평균 승차 인원"을 구하는 SQL 짜줘.

조건:
- 테이블: daily_passengers
- weekday IN ('월','화','수','목','금')만 (주말 제외)
- direction = 'board'
- 시간대별 GROUP BY
- AVG 결과는 정수로 반올림
```

### 2. 부산 부동산 — 본인 동네 5년 시세

```
bptc_realestate.db의 apt_trade 테이블로
"동래구 아파트 평균 거래금액 연도별 추이" 차트용 SQL 짜줘.

조건:
- dealAmount는 만원 단위 → 억원으로 변환 (/10000)
- deal_date에서 연도 추출 (SUBSTR(deal_date, 1, 4))
- AVG, COUNT 함께
- LIMIT 없음 (5개년만 나옴)
```

### 3. 한국 주식 — 외국인 매매 추이

```
bptc_stocks.db에서
"삼성전자 (code='005930') 최근 1개월 외국인 순매수 + 종가 추이"

⚠ "foreign"은 예약어니까 i."foreign"으로 감싸기

조건:
- investor_flow + daily_price JOIN
- code='005930' AND date >= '2026-04-01'
- ORDER BY date
```

### 4. 4-테이블 JOIN — 자치구 종합 평가

```
bptc_realestate.db에서 자치구별 종합 인프라 비교 SQL 짜줘.

각 자치구별로:
- 학원 수 (academies)
- 입시 강좌 수 (academies.ipsi_count SUM)
- 병원 수 (medical)
- 학교 수 (schools)
- 인구 (population_gu, ref_date='2026-04')
- 2025년 평균 거래가 (apt_trade, 만원→억원)

조건:
- gu_codes_realty 마스터 기준으로 LEFT JOIN
- 평균 거래가 내림차순
- 16개 자치구 전부 보여줘
```

### 5. Folium 지도 시각화

```
bptc_realestate.db의 schools 테이블에서
해운대구 학교를 Folium 지도에 표시하는 Python 코드 짜줘.

조건:
- 초등학교는 파란색, 중학교는 녹색, 고등학교는 빨간색 마커
- 학교명 popup
- HTML로 저장 (해운대_학교.html)
- 카톡으로 공유 가능한 단일 파일
```

## ❌ 자주 깨지는 프롬프트 (피하기)

### 잘못된 예 1: 컨텍스트 없이 질문
```
❌ "부산 아파트 가격 조회해줘"

이유: 테이블명·컬럼명을 AI가 추측 → 90% 틀린 SQL 생성
```

### 잘못된 예 2: 단위 누락
```
❌ "엘시티 평균 가격 알려줘"
→ AI가 dealAmount를 원 단위로 오해 → 49,800,000원 (4980만)이라고 답

✅ "엘시티 평균 거래금액 알려줘 (dealAmount는 만원 단위니까 억원으로 변환)"
```

### 잘못된 예 3: 예약어 처리 안 함
```
❌ "외국인 순매수 TOP 5 알려줘"
→ AI가 SELECT foreign 으로 짬 → SQLite syntax error

✅ "외국인(foreign 컬럼은 예약어니까 큰따옴표로) 순매수 TOP 5"
```

## 💎 강의 핵심 메시지 — 4가지 흐름

### A. **스키마 보여주기**
```
첫 줄: PRAGMA table_info(테이블명) 결과 붙여넣기
→ AI가 컬럼명·타입 정확히 인식
```

### B. **자연어 질문 + 제약**
```
질문 + LIMIT, 단위 변환, 정렬 등 제약 명시
→ 첫 시도부터 동작하는 SQL
```

### C. **결과가 맞는지 되묻기**
```
"이 결과 다시 봐줘 — 표 합치면서 부풀려진 건 아닌지,
 평균이 쏠린 건 아닌지, 단위는 맞는지 확인해줘"
→ AI가 오류 없이 틀린 답을 거르는 장치
```

### D. **결과 → 시각화 직결**
```
"이 결과를 Plotly HTML 차트로 만들어줘. 모바일에서 열려야 함"
→ 산출물 즉시 생성 (카톡 공유 가능)
```

## 🛡 자주 만나는 에러 + 해결

| 증상 | 원인 | 프롬프트로 해결 |
|------|------|--------------|
| `no such table` | 테이블명 추측 오류 | "테이블 목록 먼저 SELECT name FROM sqlite_master로 확인" |
| `no such column` | 컬럼명 추측 오류 | "PRAGMA table_info() 먼저 실행" |
| `near "foreign"` | 예약어 미처리 | "foreign 같은 예약어는 큰따옴표" 명시 |
| 결과 0행 | WHERE 조건 잘못 | "WHERE 빼고 LIMIT 10 먼저" |
| 너무 큰 결과 | LIMIT 누락 | "모든 쿼리에 LIMIT 100 기본" |
| 한글 깨짐 | 인코딩 | "결과는 UTF-8로 출력" |
