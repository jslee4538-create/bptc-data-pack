# bptc-data-pack

부산 직장인 대상 SQL · AI 활용 실습용 SQLite 데이터 패키지.

도시철도 · 부동산 · 주식 — 약 **240만 행, 23개 테이블**.

---

## 빠른 시작

### 1단계 · 클론하고 압축 풀기

```bash
git clone https://github.com/fivetaku/bptc-data-pack.git
cd bptc-data-pack/02_데이터베이스
unzip 압축/bptc_all.zip
```

압축 해제 후 생기는 파일:
```
02_데이터베이스/
├── bptc_busan.db        66 MB   부산 도시철도
├── bptc_stocks.db       83 MB   한국 주식
└── bptc_realestate.db  149 MB   부산 부동산 + 학군
```

### 2단계 · DB 구조를 먼저 읽히고 질문

`03_AI활용/CLAUDE.md`에는 이 데이터베이스에 어떤 표·컬럼이 있는지 정리돼 있습니다.
이 파일을 먼저 읽히는 건 **AI가 DB 구조를 파악하게 하기 위해서**입니다. 구조를 알아야 정확한 SQL을 짭니다.

```
"03_AI활용/CLAUDE.md를 읽고 이 데이터베이스 구조를 파악해줘.
 그다음 bptc_realestate.db에서 해운대구 2025년
 단지별 평균 거래금액 TOP 10을 보여줘."
```

질문만 하면 Claude Code가 알아서 데이터를 조회해 결과를 표·차트로 답합니다.
수강생이 SQL을 짜거나 들여다볼 필요는 없습니다 — 그 사이 쿼리는 AI가 알아서 만듭니다.

> 무엇을 물어볼지 막막하면 첫 질문을 이렇게 던져도 됩니다 —
> "이 데이터베이스로 어떤 걸 물어볼 수 있는지 예시를 알려줘."

## AI에게 시키는 흐름

이 패키지의 핵심은 **"SQL을 외우지 말고 AI에게 시킨다"**.

```
1. Claude Code 새 대화 → 03_AI활용/CLAUDE.md 읽혀 DB 구조 파악
2. 자연어로 질문 ("해운대구 5년 시세 추이 보여줘")
3. Claude Code가 알아서 데이터 조회 (쿼리는 AI가 작성)
4. 결과 → "이걸 차트 HTML로 만들어줘" 요청
```

수강생은 SQL을 직접 쓰지 않습니다. 질문을 던지고 결과를 받습니다.
자세한 프롬프트 패턴은 [`03_AI활용/프롬프트_템플릿.md`](03_AI활용/프롬프트_템플릿.md).

## 데이터 한계

- **부산 도시철도** — 1~4호선만 (동해선·부산김해경전철 제외), stations 테이블 위경도 비어있음

- **부산 부동산** — 아파트 매매·전월세만 (연립·단독 제외), 5년치 (2021-01 ~ 2026-05)
